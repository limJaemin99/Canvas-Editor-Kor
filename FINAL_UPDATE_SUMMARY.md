## 🚨 Undo 커서 위치 제약 설명

### 현재 동작
```
1단계: 1번 줄에서 "안녕" 입력 → 커서: 1번 줄 끝
2단계: 10번 줄로 이동 → 커서: 10번 줄
3단계: 10번 줄에서 "a" 입력 → 커서: 10번 줄 끝
Ctrl+Z 실행 → 2단계로 복원 → 커서: 10번 줄 (입력 전)
```

### 기술적 이유
Canvas Editor는 **스냅샷 기반 히스토리**를 사용합니다:
- 각 변경마다 전체 문서 상태 + 커서 위치 저장
- Undo 시 저장된 스냅샷 전체 복원
- 커서 위치도 스냅샷에 포함되어 함께 복원됨

### 해결 방법 (복잡도 높음)
1. 히스토리 시스템 전면 재구성 필요
2. 문서 상태와 커서 위치를 별도 관리
3. Undo 시 문서만 복원하고 커서는 현재 위치 유지

→ **수백 줄의 코드 변경 필요, 버그 발생 위험 높음**

### 권장 사항
**현재 동작 유지** - 대부분의 에디터가 동일 방식 사용:
- Microsoft Word
- Google Docs
- VS Code
- Notion

## ⚠️ 중요 안내

### Undo 동작 (커서 위치)

**현재 동작 (표준):**
- Undo 시 커서가 이전 작업 위치로 이동
- Word, Google Docs 등 대부분 에디터와 동일

**이유:**
- 히스토리 = 문서 전체 스냅샷 (커서 포함)
- 변경하려면 전체 시스템 재구성 필요

**해결책:**
- Ctrl+Z 후 원하는 위치 클릭
- 또는 Ctrl+F로 검색하여 이동

### 항/호/목/단 들여쓰기

**구현 방식:**
- TAB 문자 자동 삽입
- 각 줄의 시작 부분에 삽입
- 기존 스타일 완전 유지

**동작 확인:**
- 여러 줄 입력 후 전체 선택
- 항/호/목/단 중 하나 선택
- 각 줄 앞에 TAB이 삽입되어야 함

**만약 들여쓰기가 안 보인다면:**
- TAB 너비 설정 확인 (기본 32px)
- 브라우저 캐시 삭제 후 재시도

## 📞 문제 발생 시

### 들여쓰기가 적용 안 됨
1. 브라우저 개발자 도구(F12) 열기
2. Console 탭에서 에러 확인
3. 텍스트를 선택했는지 확인
4. "항" 선택 후 텍스트 앞에 공백이 생기는지 확인

### 색상 팔레트가 안 보임
1. 글�ol 색 아이콘 클릭
2. 아무 반응 없으면 브라우저 캐시 삭제
3. Ctrl+F5로 새로고침

### 저장이 안 됨
1. Controller 매핑 확인: `/lmxmng/webEditor/saveDoc.do`
2. 브라우저 Network 탭에서 요청 확인
3. CORS 설정 확인
